# BOT README

## ССылка на бота
@absolutelycinemabot_bot

## Описание

Этот Telegram-бот предоставляет информацию о фильмах, используя несколько источников данных, таких как **OMDb API**, **Kinopoisk API** и скрапинг сайтов. Бот принимает название фильма и ищет информацию о нем в разных базах данных, включая год, жанр, рейтинг, описание и ссылки на постеры. Бот также предоставляет ссылки на сайты для просмотра фильма, если таковые найдены.

Бот использует асинхронные запросы, чтобы ускорить процесс получения данных и поддерживает масштабируемость для добавления новых источников или API.

### Допусимые команды:
- `/start`           - Выводит приветственное окно
- `/help`            - Выводит окно помощи с допустимыми командами
- `/history`         - Выводит историю поиска: только уникальные запросы
- `/clean_history`   - Очищает историю поиска
- `/stats`           - Выводит статистику вида запрос - количество запросов. Легко добавляется любая статистика, немного изменив SQL запрос
- `<любой текст>`    - Выводит результат поиска фильмов.
- Все остальное    - Выводится заглушка, чтобы бот не упал, а сообщение было обработано.

## Структура

- **`bot/`**: Основная структура бота
  - **`/domain/`**: Содержит абстракцию для работы с базой данных
  - **`/infrastructure/`**: Инфраструктурный код, например, для работы с PostgreSQL
  - **`/middlewares/`**: Промежуточные обработчики, например, для сохранения запросов в базу
  - **`/handlers/`**: Обработчики команд, работающие с сообщениями пользователя
    - **`/search_engine/`**: Реализация API запросов, включая OMDb, Kinopoisk, и Google

Бот использует несколько внешних сервисов и библиотек (см `requirements.txt`):
- **aiogram**: Для работы с Telegram API
- **aiohttp**: Для асинхронных HTTP-запросов
- **google-api-python-client**: Для работы с Google Custom Search API

Используется база данных PostgreSQL, которая разворачивается в докере, как и само приложение (см `Makefile`, `Dockerfile`).

Телеграм-бот запущен на VPS для перебойной работы.

## Поиск при помощи API

Используется 2 вида поиска:
- По открытым базам (API Kinopoisk, OMDb API)
- Google Search

### Поиск по открытым базам
- **API_KEYs** хранятся в переменных окружения, которые подтягиваются с `load_dotenv`.
- Через **aiohttp** делается запрос специального вида, описанный в документациях: [OMDb](https://www.omdbapi.com/) и [Kinopoisk Unofficial API](https://kinopoiskapiunofficial.tech/documentation/api/#/).
- Запросы обрабатываются, и выдается результат. Это открытые базы данных, содержащие информацию о фильмах. Ссылок на просмотр тут нет.

Я специально добавил это для нахождения большего количества информации о фильме. Если выводимой информации недостаточно, можно вывести больше, которая просто отбрасывается.

### Поиск через Google Custom Search
Как корректно настраивать поиск в Google через API объясняется на сайте [Habr: как использовать Google Custom Search API](https://habr.com/ru/articles/421173/).

- Официальная библиотека **google-api-python-client** позволяет отправлять поисковые запросы.
- Отправляется запрос "Смотреть {query} онлайн бесплатно", где `query` — это название фильма, введенное пользователем.
- Проверяются полученные ссылки на корректность (пытаемся подключиться к ним, часто они битые из-за РАЗНЫХ причин). Из-за этого времени отклика бота может увеличиться. Если убрать проверку ссылок, то бот будет реагировать очень быстро, но видео будет недоступно.
- Для показа описания выбирается первый доступный сайт, остальные ссылки выводятся после описания.

### Примечание
- Если использовать только базу данных о фильмах, тогда не будет ссылок на видео, а также некоторые фильмы могут не отобразиться.
- Если использовать поиск, фильм может отобразиться некорректно, но это уже зависит от введенного запроса пользователя.
- Предусматривается, что запрос корректно отображает название фильма. Идеально, если в нем нет ошибок, и название идеально соответствует.
- Более полная информация о фильме показывается из баз данных, когда ссылки — из поиска.
